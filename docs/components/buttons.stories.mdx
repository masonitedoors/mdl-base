# Buttons

We include several predefined button styles, each serving its own semantic purpose, with a few extras thrown in for more control.

## Button tags

The `.btn` classes are designed to be used with the `<button>` element. However, you can also use these classes on `<a>` or `<input>` elements (though some browsers may apply a slightly different rendering).

When using button classes on `<a>` elements that are used to trigger in-page functionality (like collapsing content), rather than linking to new pages or sections within the current page, these links should be given a role="button" to appropriately convey their purpose to assistive technologies such as screen readers.

<Canvas>
  <Story name="Button Tags">
    {`
      <a class="btn btn--primary" href="#" role="button">Primary Link</a>
      <button class="btn btn--primary" type="submit">Primary Button</button>
      <input class="btn btn--primary" type="button" value="Primary Input">
      <input class="btn btn--primary" type="submit" value="Primary Submit">
      <input class="btn btn--primary" type="reset" value="Primary Reset">
    `}
  </Story>

## Disabled state

Make buttons look inactive by adding the disabled boolean attribute to any `<button>` element.

<Canvas>
  <Story name="Disabled">
    {`
      <button class="btn" disabled>Disabled Button</button>
      <button class="btn btn--primary" disabled>Disabled Primary Button</button>
      <button class="btn btn--secondary" disabled>Disabled Secondary Button</button>
    `}
  </Story>
</Canvas>

Disabled buttons using the `<a>` element behave a bit different:

- `<a>`s don’t support the disabled attribute, so you must add the `.btn--disabled` class to make it visually appear disabled.
- Disabled buttons should include the aria-disabled="true" attribute to indicate the state of the element to assistive technologies.

<Canvas>
  <Story name="Disabled Link">
    {`
      <a href="#" class="btn btn--disabled" tabindex="-1" role="button" aria-disabled="true">Disabled Link</a>
      <a href="#" class="btn btn--primary btn--disabled" tabindex="-1" role="button" aria-disabled="true">Disabled Primary Link</a>
      <a href="#" class="btn btn--secondary btn--disabled" tabindex="-1" role="button" aria-disabled="true">Disabled Seconary Link</a>
    `}
  </Story>
</Canvas>

**Link functionality caveat**

The `.btn--disabled` class uses `pointer-events: none` to try to disable the link functionality of `<a>`s, but that CSS property is not yet standardized. In addition, even in browsers that do support `pointer-events: none`, keyboard navigation remains unaffected, meaning that sighted keyboard users and users of assistive technologies will still be able to activate these links. So to be safe, add a `tabindex="-1"` attribute on these links (to prevent them from receiving keyboard focus) and use custom JavaScript to disable their functionality.
